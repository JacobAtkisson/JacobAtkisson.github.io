Predictive Maintenance with Machine Learning

AI4I 2020 Dataset – Classification Model Evaluation

View rendered HTML version

Overview

This project applies supervised machine learning techniques to a real-world predictive maintenance problem. Using the AI4I 2020 dataset from the UCI Machine Learning Repository, I built and evaluated multiple classification models to predict machine failure events based on operational sensor data.

The focus of this project is model comparison, parameter tuning, and evaluation, demonstrating a full end-to-end ML workflow.

Objective

To accurately predict whether a machine will fail using sensor and operational data, enabling proactive maintenance and reduced downtime.

Dataset

  • Source: UCI Machine Learning Repository – AI4I 2020 Predictive Maintenance Dataset
  • Instances: 10,000
  • Features:
    • Air temperature
    • Process temperature
    • Rotational speed
    • Torque
    • Tool wear
  • Target Variable: Machine failure (binary classification)

Minimal preprocessing was required, allowing emphasis on modeling and evaluation techniques.

Methods & Models

The following machine learning techniques were implemented and compared:

  • Decision Tree Classifier
    • Tuned maximum depth to balance bias and variance
  • k-Nearest Neighbors (k-NN)
    • Evaluated impact of varying k
  • Principal Component Analysis (PCA)
    • Used to reduce dimensionality and analyze performance trade-offs
  • Train/Test Splitting
    • Ensured unbiased evaluation

Evaluation Metrics

Models were evaluated using:

  • Accuracy
  • Confusion matrices
  • Comparative performance analysis

This allowed clear identification of model strengths, weaknesses, and overfitting behavior.

Key Results

  • Decision Trees achieved the highest overall accuracy when properly depth-limited
  • k-NN performance was sensitive to the choice of k
  • PCA reduced dimensionality but introduced a small trade-off in accuracy
  • Model tuning significantly impacted generalization performance

Skills Demonstrated

  • Python (pandas, NumPy)
  • scikit-learn (classification, PCA, model evaluation)
  • Machine learning model selection & tuning
  • Bias–variance tradeoff analysis
  • Data-driven decision making

Library Database Program

This program acts as a tool to update, search, and add to an existing library DB.

Usage

Compile the program using: javac -cp mysql-connector-j-9.5.0.jar *.java

Run using command: java -cp '.\;mysql-connector-j-9.5.0.jar' Main

Once you've run the program will start connecting to the Database. After it has connected you will see this menu:

===== LIBRARY MANAGEMENT SYSTEM =====
1. Add Author
2. Add Book
3. View All Authors
4. Search Books
5. View Books by Author
6. Update Author
7. Update Book
8. Delete Author
9. Delete Book
0. Exit

You may select any option and fillout the prompts as follows and the database will be updated.
